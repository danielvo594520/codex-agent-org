# Output Formats

各 agent が返す報告フォーマットです。
Leader が差分確認と採用判断をしやすいように、短く具体的に書きます。

## Explorer Report

```md
調査結果:
- ...

関連ファイル:
- ...

影響範囲:
- ...

実装ギャップ:
- ...

不明点:
- ...
```

## Worker Report

```md
実施内容:
- ...

変更ファイル:
- ...

確認:
- `...`

残リスク:
- ...
```

## Reviewer Report

```md
指摘:
- [重要度] `file:line` 内容

テスト不足:
- ...

仕様確認が必要な点:
- ...

総評:
- 採用可 / 修正後採用 / 採用不可
```

## Integrator Report

```md
統合結果:
- ...

取り込んだ変更:
- ...

想定外ファイル:
- なし / ...

確認:
- `git status`
- `...`

残リスク:
- ...
```

## Claude Code Agent Report

```md
実施内容:
- ...

参照・変更ファイル:
- ...

テスト結果:
- ...

判断に迷った点:
- ...

Codex Leader が確認すべき点:
- ...
```

## Leader Final Report

```md
完了しました。

作成・更新したファイル:
- ...

確認:
- ...

残リスク:
- ...

次にやるなら:
- ...
```
