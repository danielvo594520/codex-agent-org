# Safety Rules

複数エージェント運用で守る安全ルールです。
速度よりも、確認可能性と復旧可能性を優先します。

## 禁止事項

- `git reset --hard`
- `git checkout -- .`
- secret の表示
- 本番環境操作
- 自動 push
- 自動 deploy
- 担当外ファイルの編集
- 他 agent の変更 revert
- ユーザー未承認の大規模削除
- ユーザー未承認の release

## Claude Code Agent に渡さない作業

- 秘密情報を含む調査
- 本番データを扱う作業
- 認証・権限の最終判断
- 課金・削除・通知など副作用が大きい操作
- GitHub release や deploy
- ユーザー確認が必要な意思決定

## 変更前の確認

作業開始前に Leader は以下を確認します。

- 対象ファイル
- 既存パターン
- 現在の `git status`
- 他 agent の担当範囲
- 作業の完了条件

## 変更中のルール

- 小さく変更する
- 担当範囲を守る
- 不明点は仮定として明記する
- 他 agent の差分を消さない
- テストや確認コマンドを記録する

## 採用前の確認

Leader は成果物を採用する前に以下を確認します。

```bash
git status
git diff --stat
git diff
```

必要に応じてテスト、lint、ビルド、ドキュメント確認を行います。

## 失敗時の扱い

- 失敗ログを保存する
- 原因を切り分ける
- 変更範囲を確認する
- 不要な修正を重ねない
- 破壊的な巻き戻しはしない

## 最終判断

エージェントの成果物は候補です。
採用、修正、破棄の判断は必ず Codex Leader が行います。
