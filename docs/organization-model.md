# Organization Model

Codex Leader を中心に、役割を分けて複数エージェントを運用するための組織モデルです。

## Leader

責務:

- ユーザー要求を目的、制約、完了条件に分解する
- 作業を Explorer / Worker / Reviewer / Integrator / Claude Code Agent に割り当てる
- 成果物、diff、テスト結果、残リスクを確認する
- 採用、修正、破棄を最終判断する

任せてよい作業:

- すべての作業の整理、統合、判断
- ブロッキング作業の直接対応
- 最終報告

任せてはいけない作業:

- 検証していない成果物の自動採用
- ユーザー未承認の破壊的操作
- 自動 push / deploy

期待する出力:

- 作業計画
- 担当割り当て
- 採用判断
- 確認結果と残リスク

## Explorer

責務:

- コードベースや仕様を調査する
- 関連ファイル、既存パターン、影響範囲を特定する
- 実装ギャップや不明点を報告する

任せてよい作業:

- 調査
- ファイル一覧化
- 既存実装の要約
- 影響範囲の整理

任せてはいけない作業:

- コード変更
- 仕様の最終決定
- git 操作

期待する出力:

- 調査結果
- 関連ファイル
- 影響範囲
- 実装方針の候補
- 未確認事項

## Worker

責務:

- 指定された範囲だけを実装する
- 必要なテストやドキュメントを追加する
- 変更ファイルと確認結果を報告する

任せてよい作業:

- 限定されたファイルの変更
- 小さな修正
- テスト追加
- ドキュメント草案作成

任せてはいけない作業:

- 担当外ファイルの編集
- 他 agent の変更 revert
- 大規模リファクタ
- 自動 commit / push / deploy

期待する出力:

- 実施内容
- 変更ファイル
- 実行した確認コマンド
- 残リスク

## Reviewer

責務:

- 差分をレビューする
- バグ、リグレッション、テスト不足、セキュリティリスクを指摘する
- 仕様との不一致を重要度順に報告する

任せてよい作業:

- diff レビュー
- テスト観点の確認
- リスク洗い出し
- 採用前の品質確認

任せてはいけない作業:

- 差分の直接修正
- 称賛中心のレビュー
- 未確認の断定

期待する出力:

- 重要度順の指摘
- 根拠となるファイルや行
- テスト不足
- 残リスク

## Integrator

責務:

- 複数成果物を統合する
- 変更範囲、衝突、想定外ファイルを確認する
- テスト実行と採用判断を補助する

任せてよい作業:

- diff 確認
- 統合順の整理
- 衝突解消案の作成
- 検証コマンドの実行補助

任せてはいけない作業:

- 未検証変更の採用
- 勝手な仕様変更
- 自動 push / deploy

期待する出力:

- 統合結果
- 想定外変更の有無
- 検証結果
- 採用判断の材料

## Claude Code Agent

責務:

- Claude Code CLI 経由で独立した調査、草案作成、実装候補、レビューを行う
- Codex とは別視点の提案を返す
- 指定された作業ディレクトリと許可ファイルだけで作業する

任せてよい作業:

- 独立調査
- ドキュメント草案
- 代替実装案
- レビュー観点の作成

任せてはいけない作業:

- 秘密情報を扱う作業
- 本番環境操作
- git push / release / deploy
- 破壊的操作
- ユーザー確認が必要な意思決定

期待する出力:

- 実施内容
- 変更または参照したファイル
- テスト結果
- 判断に迷った点
- Codex Leader が確認すべき点
